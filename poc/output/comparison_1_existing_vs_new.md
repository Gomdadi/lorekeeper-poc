# 비교 ① 기존(run1) vs 신규(run2) — 재현성/편차

> 같은 설정을 한 번 더 돌려 run-to-run 편차를 본다. luna 고정.
> 원본: run1 = `luna_medium.md`/`luna_high.md`, run2 = `luna_medium_run2.md`/`luna_high_run2.md`.

## medium: run1 vs run2

| 항목 | run1 (기존) | run2 (신규) |
|---|---|---|
| Event | 10 | 6 |
| Item | 2 (멸살법 + **작가의 특별한 선물**) | 1 (멸살법만) |
| 선물 | Item만 생성, **소유·관계 없이 고립** | **없음** |
| 자전거 | 없음 | 없음 |
| CharacterState 중복 | **있음** (김독자 소속×2·신분×2) | **없음** |
| Organization | 2 (대기업 계열사, **인사팀**) | 1 (대기업 계열사) |
| 유상아 소속 | 인사팀 | 대기업 계열사 (부서 뭉갬) |
| 동료 관계 | ✓ | ✓ |
| story_order(2화) | 2.0~2.5 | 2.0~2.2 |

→ **편차 큼**: run1은 선물 Item·중복·인사팀·10 event, run2는 선물 없음·중복 없음·대기업만·6 event. 같은 설정인데 결과 구성이 상당히 다르다.

## high: run1 vs run2

| 항목 | run1 (기존) | run2 (신규) |
|---|---|---|
| Event | 9 | 7 |
| Item | 3 (멸살법, 첨부파일, **자전거**) | 2 (멸살법, 첨부파일) |
| 자전거 | **✓ 소유=상실 +ABOUT** | **없음** |
| 선물(첨부파일) | ✓ 소유=보유 +ABOUT | ✓ 소유=보유 +ABOUT |
| Organization | 1 (대기업 계열사) | 2 (대기업 계열사, 인사팀) |
| 유상아 소속 | 대기업 계열사 | **대기업 계열사 + 인사팀(중복)** |
| 동료 관계 | 없음 | ✓ |
| story_order 승진 | 1.9(과거) | 1.8(과거) |

→ **high도 편차**: 자전거는 run1만, 동료는 run2만, Org 1↔2, 유상아 소속 중복은 run2에서 발생. 다만 **선물(소유=보유)은 두 run 모두 잡음**(high의 상대적 강점).

## 종합
- **두 조건 다 run-to-run 편차가 큼.** "자전거+선물 동시 완전 추출"은 어느 run도 안정적으로 재현 못 함(high run1만 둘 다).
- 재현되는 것: 세카이 배제, 일시적 상태 배제, 소속 reified, story_order 증가 — **핵심 규칙은 안정**. 흔들리는 건 **한 번 언급 사실(자전거·선물)의 recall과 부서 입도·중복**.
- 단발 표본으로 우열/개선을 단정할 수 없음을 재확인. **N회(3~5) 반복 집계가 필요**.
