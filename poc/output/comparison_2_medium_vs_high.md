# 비교 ② medium vs high — 신규 run2 (동일 시점 실행)

> luna 고정, 같은 프롬프트로 medium·high를 연달아 1회씩 실행한 결과 비교.
> 원본: `luna_medium_run2.md`, `luna_high_run2.md`.

## 토큰·비용

| | 입력 | **출력** | 합 | **비용** |
|---|---|---|---|---|
| medium | 25,009 | 5,673 | 30,682 | **$0.0590** |
| high | 25,054 | 15,239 | 40,293 | **$0.1165** |
| 배수 | ~1.0x | **~2.7x** | ~1.3x | **~2.0x** |

- 입력 동일, 비용은 출력(추론) 토큰이 결정. high가 약 **2배 비용**.

## 그래프

| 항목 | medium run2 | high run2 |
|---|---|---|
| Event | 6 | 7 |
| Item | 1 (멸살법) | 2 (멸살법, **첨부 파일**) |
| **선물(첨부파일)** | ✗ 없음 | ✅ `김독자 소유=보유 -ABOUT-> 첨부 파일` |
| 자전거 | ✗ 없음 | ✗ 없음 |
| CharacterState | 4 (중복 없음) | 6 (중복 없음) |
| Organization | 1 (대기업 계열사) | 2 (대기업 계열사, 인사팀) |
| 유상아 소속 | 대기업 계열사 | 대기업 계열사 **+ 인사팀(중복)** |
| 동료 관계 | ✓ | ✓ |
| story_order 승진 | 별도 사건 없음(2.0~2.2) | **1.8(과거 정확)** |
| 세카이/일시적 상태 배제 | ✅ | ✅ |

## 분석
- **high가 잡는 것**: 선물(첨부파일) 소유 상태, 조직 2개, 과거 승진 시간 배치. medium은 선물을 통째로 놓침.
- **high의 이번 결함**: 유상아 소속이 대기업·인사팀 **둘 다** 생성(중복). medium은 대기업 하나(부서 뭉갬).
- **둘 다 자전거 누락**(이번 두 run 공통).
- **비용**: medium이 high의 절반.

## 결론
- 이번 동시 실행에서도 **high가 구조 완전성(선물 소유·조직·시간순)에서 우위**, medium은 절반 비용.
- 단 high가 완벽하진 않음(소속 중복, 자전거 누락) — 그리고 비교 ①에서 봤듯 **둘 다 편차가 커서** 이 우열이 매번 재현된다는 보장은 없다.
- **권고**: 소지품 소유 같은 검증 대상 구조가 중요하면 high(비용 2배 감수), 비용 우선이면 medium. 다만 어느 쪽이든 **자전거·선물 recall 편차는 다중 추출(N회 투표/합집합)**로 잡아야 안정적.
